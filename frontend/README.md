# Система учета заявок (Fullstack MVP)

Небольшое приложение для учета внутренних заявок, разработанное в рамках тестового задания.

## 🛠 Стек технологий

* **Backend:** Python, FastAPI, SQLAlchemy, SQLite
* **Frontend:** React, TypeScript, Vite, Axios
* **Архитектура:** Раздельные клиент и сервер, взаимодействие по REST API

## ✨ Реализованный функционал

* Создание новых заявок (с валидацией полей).
* Просмотр списка с поддержкой серверной пагинации.
* Серверная фильтрация (по статусу и приоритету) и поиск (по названию и описанию).
* Серверная сортировка (по дате создания и приоритету, по убыванию/возрастанию).
* Изменение статуса заявок (с блокировкой статуса `done`).
* Авторизация администратора (`HTTPBasic`).
* Удаление заявок (доступно только администратору).

## 🚀 Инструкция по запуску локально

Проект состоит из двух независимых частей. Для запуска потребуется открыть два терминала.

### Шаг 1: Запуск Backend (FastAPI)

При первом запуске автоматически будет создана база данных SQLite (`tickets.db`). Сервер запустится на `http://localhost:8000`.

1. Перейдите в папку бэкенда:
   ```bash
   cd backend

    Создайте и активируйте виртуальное окружение:

        Для Windows:
        code Bash

        python -m venv venv
        venv\Scripts\activate

        Для macOS/Linux:
        code Bash

        python -m venv venv
        source venv/bin/activate

    Установите зависимости:
    ```Bash

    pip install fastapi uvicorn sqlalchemy pydantic

    Запустите сервер разработки:
    ```Bash

    uvicorn main:app --reload

    Интерактивная документация API (Swagger) будет доступна по адресу: http://localhost:8000/docs

Шаг 2: Запуск Frontend (React)

Клиент запустится на http://localhost:5173.

    Откройте новый терминал и перейдите в папку фронтенда:
    ```Bash

    cd frontend

    Установите зависимости:
    ```Bash

    npm install

    Запустите сервер разработки:
    ```Bash

    npm run dev

💡 Технические решения и упрощения

В рамках данного тестового задания (согласно условию «Дизайн не оценивается») были приняты следующие архитектурные компромиссы для ускорения разработки:

    Авторизация и подтверждения: Вход администратора и подтверждение удаления реализованы через нативные браузерные методы prompt() и confirm(). В реальном production-приложении для предотвращения блокировки всплывающих окон браузером необходимо использовать кастомные React-модальные окна с управлением через глобальный стейт.

    Стилизация: Использованы инлайн-стили (style={{...}}) для минимизации зависимостей. Для масштабируемого проекта целесообразно использование Tailwind CSS или CSS-модулей.

    Оптимизация: Добавлен механизм debounce (задержка) для текстового поиска, чтобы предотвратить излишнюю нагрузку на сервер при быстром вводе, а также внедрена блокировка кнопок пагинации на время ожидания ответа от API.
